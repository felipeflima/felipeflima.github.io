Challenge 5: Accessibility Concerns

No changes in Challenge 5

I believe that "Challenge 2: Building a Chessboard" might be the one that presents the greatest acessibility issue this assignment. The use of the table element to create a visual representation of a chessboard makes it so a screenreader would have trouble conveying the information in form of sound and if not properly coded would result in near gebrish to a listner. To resolve this, proper semantic markup should be used by making sure HTML elements are being used according to their intended purpose and creating a separation of content and presentation.

Another issue could be the color contrast between the chessboard and the chess pieces. If the contratst is not large enough, it could cause some readers to have a difficult time understanding the board, locating and identifiying the pieces. To resolve this issue the simplest way I created a standard black and white board and set the pieces to be red, so it would contrast with both colors of the board.
